Welcome to the Ring Signature System

This platform lets you create and verify RSA-based ring signatures using a fully working, mathematically correct implementation based on the original "How to Leak a Secret" paper by Ronald L. Rivest, Adi Shamir and Yael Tauman.

How to Create a Signature

  1. Enter your message (text) or upload a PDF file (content will be hashed)
  2. Add ring members by clicking "Add Member" button
  3. Fill in member details: name, organization, and RSA public key (PEM format)
  4. OR use the GitHub lookup feature to automatically fetch public keys from a GitHub username
  5. Select the actual signer from the dropdown of ring members
  6. Provide the signer's RSA private key (PEM format, 1024/2048/4096-bit)
  7. Submit to generate the ring signature with consistent x-value lengths
  8. Copy the verification URL to share the signature

How to Verify a Signature

  1. Enter the original message (text) or upload the PDF file
  2. Provide the glue value (v) from the signature
  3. Enter all x-values (Base64 encoded) from the signature
  4. Provide all RSA public keys (PEM format) of the ring members
  5. Submit to verify the signature using the original nested encryption combining function

Current Implementation Features

Cryptographic Features
  • RSA-based algorithm with combining function
  • SHA-256 message hashing
  • Symmetric encryption E_k(x) = (x + k) mod 2^128
  • PEM format support for keys (1024/2048/4096-bit)
Technical Features
  • Real-time form validation with visual feedback
  • GitHub key lookup for public keys
  • PDF file support (content hashing)
  • Copy verification URL for easy sharing
  • No database dependencies
Complete Algorithm Overview
  1. Calculate k = H(m) using SHA-256 cryptographic hash
  2. Pick random glue value v
  3. Compute temporary x_signer to determine target length for anonymity protection
  4. Generate random x_i for non-signers with matching lengths and compute y_i = g_i(x_i)
  5. Solve ring equation C_k,v(y_1, y_2, ..., y_n) = v by working backwards through nested encryption
  6. Calculate final x_signer = g_signer^-1(y_signer) using private key
